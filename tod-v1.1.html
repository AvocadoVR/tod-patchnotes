Changelog - v1.1 / v1.1.1

v1.1 - Initial Release

New

  • Added <local> placeholder that returns the local player (the user who clicked the Truth or Dare button).
  • Added <range[1-2]> placeholder that generates any positive numbers within a set range (maximum: 500).

Changed

  • Modified <player> functionality: now uses <player=X> where X = 1-5 (up to 5 players supported).
    • Example: <local> do you have a crush on any of these players: <player=1>, <player=2>, <player=3>
  • Increased download rate limit from 10 seconds to 15 seconds for sets.

Fixes

  • Fixed a second RequestSerialization call that was unnecessary.

Removed

  • Removed Player Truths & Player Dares.
  • Removed SetQuestion() and Chance() functions.
  • Removed usePlaceholders — placeholders are now allowed by default.

v1.1.1

Fixes

  • Fixed a critical bug where clicking the Truth or Dare button before the "Ready!" message would break the game.
  • Resolved an issue with GetAllPlayers not correctly populating the local player array.
  • Fixed player placeholder handling: unused placeholders (e.g., <player=5>) are now correctly removed if there aren’t enough players.
  • Corrected the input range limit to consistently stay within 1–500.
Critical Bug in v1.1: If you clicked the button before the "Ready!" message appeared, the game could break. This is fixed in v1.1.1.
Note: The v1.1.1 patch also includes fixes from the original v1.1 release and addresses the major bug where placeholders were not removed properly in certain scenarios.
